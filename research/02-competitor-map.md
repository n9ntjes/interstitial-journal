# Competitor Map

## Summary

The competitor set is broad, but the direct competitor is rare.

Most tools are built around one of these jobs:

- Plan the future.
- Track the past.
- Block distractions.
- Store notes.
- Record everything.
- Launch commands faster.

This product should own:

**Preserve and recover the user's current working thread.**

## Positioning Gap

The strongest open lane:

**Intentional memory, not surveillance.**

This product can sit between manual notes and passive screen recording:

- Manual enough to capture intent.
- Automatic enough to reduce friction.
- Visible enough to act as a HUD.
- Private enough to earn trust.

## Differentiation Pillars

### 1. Re-entry, Not Productivity Scoring

Avoid the shame loop of "you were distracted." The product should say:

"Here is the path back."

### 2. Present-Tense Capture

Planners ask "what will you do?" Journals ask "what happened?" This app asks:

"Where are you right now, and what is the next step?"

### 3. Hotkey Muscle Memory

The killer UX is not a dashboard. It is a global hotkey that appears faster than resistance can build.

### 4. Visible Breadcrumbs

The live overlay should become the recognizable artifact: the current thread always has a visible path.

### 5. Optional Context Metadata

App name, window title, repo, branch, browser domain, calendar event, or document name can enrich entries if enabled. Do not require screenshots or keystroke capture.

## Products To Watch

- Raycast: if it adds a deeper work-journal extension, it could absorb some hotkey workflow.
- Rize: if it adds intent notes and re-entry cards, it could become close.
- Tiimo: if it becomes more desktop-native and live-work oriented, it could compete for ADHD users.
- Microsoft Recall: if users trust it, OS-level memory could normalize "what was I doing" search.
- Screenpipe: local AI memory is a strong technical neighbor, but likely too heavy for the simple attention HUD wedge.

## Takeaway

Do not call this "a journaling app" in isolation. That puts it beside Day One and Reflect, where it looks too small.

Do not call it "an ADHD productivity app" in isolation. That puts it beside Tiimo, Structured, and Llama Life, where users expect planners, routines, and coaching.

Call it:

**A desktop HUD for never losing the thread.**
